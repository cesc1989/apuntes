# Deploy First

_Enviado el Jul 25, 2025_

Basicamente, es que al iniciar un proyecto nuevo el entorno de producción debe ser de las primeras cosas que se configure.

Si no hay prod no hay donde mostrar el valor generado por el software construido. Además, se pueden resolver dudas sobre el sistema a medida que se puede ir desplegando.

> Why deploy first? ==The answer is because no part of a software system has any value until it's working in production==. Until there's a production environment, there's no possibility of delivering any value.
>
> Next time you start a new project, I encourage you to set up the production environment right from the start. ==If setting up the production environment seems hard and mysterious, that's not a reason to do it later, that's a reason to do it sooner==. You can't start delivering value until you do.
