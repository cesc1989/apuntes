# Preventing bugs is the start

_Enviado el Oct 3, 2025_

La lectura trata sobre como tener una buena base de pruebas, buenos tests, permite, además de sacar errores, refactorizar de manera más agresiva.

> You may also be familiar with the idea that tests can help improve the design of your code

Al final, la ventaja de poder refactorizar de forma más contundente es prevenir que la calidad del código decaiga con el paso del tiempo.

> Refactoring is necessary because all code (...) is susceptible to "rotting".

Finalmente, refatorizar es la forma en que podemos hacer nuestro código mucho mejor en cada oportunidad. Porque al poder refactorizar podremos ver otras nuevas formas de escribir la misma funcionalidad.

> Refactoring is like traversing a range of mountains. Once you arrive at a peak, you can see a new, higher peak that you couldn't see before.
